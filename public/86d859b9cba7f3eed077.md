ウィンドウサイズが`Math.Min(rcv_wnd, Kcp.FRG_MAX)`に制限されている（ 固定値で Kcp.FRG_MAX = 255 ）ので，それを超えることはできなさそう[1] で，最大値は298449 bytes (291KB)となっています．これはSyncListをN個持った場合でも，(N x SyncListの配列数)のバイト数 < 298449 bytes (291KB)なので，それが同期の限界っぽいです．

UNETの場合は，(N x SyncListの配列数)のバイト数 < 64 KB bytes となっており（30KBとかでも適切なエラーが出ずにバグるが），明らかにMirrorで同期できるバイト数が多そうです（4.54倍）．
